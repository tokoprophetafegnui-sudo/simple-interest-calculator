# Contributing to Simple Interest Calculator

Thank you for your interest in contributing to the Simple Interest Calculator project! We welcome and appreciate all contributions from the community.

## Types of Contributions We Welcome

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome. This includes:

* **Bug Reports**: If you find a bug, please open an issue describing the problem, how to reproduce it, and any error messages.
* **Bug Fixes**: Feel free to submit pull requests that fix existing bugs.
* **Documentation Improvements**: Help us improve our documentation, README, comments, or guides.
* **Enhancements**: We're open to new features and improvements to the existing functionality.
* **Ideas**: Have a great idea? Share it by opening an issue for discussion.

## Getting Started

1. **Fork the repository**: Click the "Fork" button on GitHub to create your own copy of the project.

2. **Clone your fork**:
   ```bash
   git clone https://github.com/YOUR_USERNAME/simple-interest-calculator.git
   cd simple-interest-calculator
   ```

3. **Create a new branch**:
   ```bash
   git checkout -b your-branch-name
   ```

4. **Make your changes**: Edit files, add features, fix bugs, or improve documentation.

5. **Test your changes**: Ensure your changes work correctly and don't break existing functionality.

6. **Commit your changes**:
   ```bash
   git add .
   git commit -m "Describe your changes clearly"
   ```

7. **Push to your fork**:
   ```bash
   git push origin your-branch-name
   ```

8. **Create a Pull Request**: Go to the original repository and click "New Pull Request". Provide a clear description of your changes.

## Guidelines

* **Code Style**: Follow consistent coding practices. Keep code clean and readable.
* **Commit Messages**: Write clear, descriptive commit messages that explain what and why you made changes.
* **Testing**: Test your changes thoroughly before submitting a pull request.
* **Documentation**: Update documentation and comments as needed to reflect your changes.
* **Respect the Codebase**: Follow the existing code structure and patterns in the project.

## Pull Request Process

1. Ensure your pull request has a clear title and description.
2. Link any related issues in your pull request description.
3. Provide examples or test cases for new features.
4. Be responsive to feedback and review comments.
5. Ensure all checks and tests pass before merging.

## Code of Conduct

Please note that we have a [Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project, you agree to abide by its terms. We are committed to providing a welcoming and inclusive environment for all contributors.

## Reporting Issues

When reporting issues, please include:

* A clear, descriptive title
* A detailed description of the problem
* Steps to reproduce the issue
* Expected behavior vs. actual behavior
* Any error messages or logs
* Your operating system and environment details

## Questions?

Feel free to open an issue if you have questions about contributing or need clarification on anything.
